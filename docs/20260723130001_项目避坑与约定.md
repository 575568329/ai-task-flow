# 项目避坑与约定（运行时）

> 本文档汇总 ai-task-flow 项目里**反复坑过人**的规律性事项。来源：git 历史 fix 提交、代码踩坑注释、.gitignore。
> 依赖安装相关坑见 [20260723130000_依赖安装排坑记录.md](20260723130000_依赖安装排坑记录.md)。
> CLAUDE.md 第十节仅保留红线清单，细节全部在此。

---

## 一、端口与启动顺序

- 前端 dev 固定 **`http://localhost:5678`**，`/api` 代理到 backend；backend 默认 **3000**，被占会自动顺延（commit `d5012d0`）。**禁止同时起多个 `npm run dev`**。
- **vite 代理坑**：本机其他服务（如 Astro）占用同一端口会导致 `/api` 全 404。排查 `/api` 异常先确认端口独占。（`frontend/vite.config.ts`）
- dev 有**严格启动顺序**：`shared` 先构建 → `backend` → `frontend`，脚本内有就绪等待。**不要跳过单独起 frontend**（shared 未构建会报找不到 `@ai-task-flow/shared`）。

---

## 二、后端 DI（tsyringe）

- MCP server 的 `TaskRepository` **必须用 `useFactory` 注册**，直接绑定实现会导致启动 DI 崩溃（commit `269a920`）。新增需 DI 的仓储/服务时照此办理。

---

## 三、任务状态机（会话化改造后）

- 状态收敛为**三态：`TODO` / `DONE` / `BLOCKED`**。打开终端**不改状态**，`TODO` 可直接回写结果，**不再走「派发→审核」两段式**。（`backend/src/domain/workflow/entities/Task.ts`、`value-objects/TaskStatus.ts`）
- 旧 `tasks.json` 里的 `dispatched` / `review` 状态加载时自动归一为 `TODO`，且只归一一次。（`backend/src/infrastructure/persistence/JsonTaskRepository.ts`）
- 任务前缀非必填，空/非法值回落 `TASK`（commit `d8c6134`）。

---

## 四、安全脱敏（重要）

- `LlmConfigService` 的返回值**含 API Key**，**禁止透传给前端或写入日志**；前端展示必须用 `getMaskedConfig()`。（`backend/src/application/llm-config/LlmConfigService.ts`）

---

## 五、跨进程同步

- MCP 回写 `tasks.json` 后，前端靠**文件轮询**刷新（非实时推送，commit `e82a109`）。回写后等几秒再看板，不要误判为「没生效」。

---

## 六、运行时产物（勿手动提交，已 gitignore）

- **`*hook-events.jsonl`**：看板程序 kanban-code 会把绝对路径当文件名反复写进项目目录，看到**不要提交**。
- **`backend/public/`**：前端 build 产物，由 `prepublishOnly` 发布时现建，**勿手动改/提交**。
- **`backend/uploads/`**：用户上传图片运行时产物。

---

## 七、浏览器扩展（MV3）访问 localhost

- Chrome **PNA（Local Network Access）只拦预检请求**；扩展 POST 到 backend 改用 `text/plain`（简单请求）绕过，把 fetch 挪到 service worker **不能**免除 PNA。
- CORS：带 `credentials` 时 `origin` 不能是 `*`；本项目跨域调用方都不带 cookie，设 `credentials: false`。
- 图片上传：Fastify `bodyLimit` 已提至 25MB（默认 1MB 会 413）；跨域 fetch 的 `blob.type` 不可靠，落盘不要硬依赖 mime 前缀。
- 完整四层坑（CORS/PNA/413/mime）见 [20260617020000_网页剪藏扩展联调踩坑记录.md](20260617020000_网页剪藏扩展联调踩坑记录.md)。

---

## 八、前端组件已知坑

- **图片预览**：手写蒙版在层级/居中/点击穿透上反复踩坑，已弃用；改用 YARL，注意其蒙版 portal 到 `document.body`、不在 Radix Dialog 树内。（`frontend/src/components/ui/image-preview.tsx`）
- **TTS**：Chromium `speechSynthesis` 有已知 bug，封装时已规避，改动前先看 `frontend/src/lib/speech.ts` / `extension/src/sidepanel/utils/speech.ts`。

---

## 九、终端启动

- `TerminalLauncher` 用 `start` 启动的终端是 **fire-and-forget**，Node **无法向其注入消息**；需要交互应走 MCP 协议而非向终端塞命令。（`backend/src/infrastructure/system/TerminalLauncher.ts`）

---

## 十、诊断方法论（来自剪藏联调）

- **`Failed to fetch` 不一定是网络问题**——CORS / PNA 都表现为这个。第一刀看**后端日志有没有收到请求**。
- **跨上下文系统，关键链路日志必须落在后端可查处**（写文件 + 镜像 console），不要只打在网页/SW 控制台或会被 pino 级别过滤的 `request.log.info`。
