## 三、技术方案

### 3.1 后端(DDD 分层,新增 knowledge 上下文)

```
backend/src/
├── domain/knowledge/
│   ├── entities/KnowledgeDoc.ts        # 文档实体(path/title/tags/mtime/links/backlinks)
│   └── value-objects/KnowledgeManifest.ts # 目录树 + 索引聚合
├── application/knowledge/
│   └── KnowledgeService.ts             # 扫描、构建 manifest、读文档、建链接索引
├── infrastructure/knowledge/
│   ├── KnowledgeScanner.ts             # fs 递归扫描 + gray-matter + 链接解析
│   └── KnowledgeWatcher.ts             # chokidar 监控 → EventBus 发事件(防抖)
└── interfaces/http/routes/
    └── knowledgeRoutes.ts              # REST API
```

**数据根目录**:在 `dataDir.ts` 新增 `knowledgeDirPath()` → `~/.ai-task-flow/knowledge/`。

**API 契约**:

| 方法 | 路径 | 入参 | 出参 | 说明 |
|------|------|------|------|------|
| GET | `/api/knowledge/manifest` | — | `{ tree, flatDocs, tags, backlinks }` | 目录树 + 扁平列表 + 标签全集 + 反向链接索引 |
| GET | `/api/knowledge/doc?path=` | path | `{ path, kind, content?, meta }` | 读单篇(md 返回文本;二进制返回元信息,前端走 raw) |
| GET | `/api/knowledge/raw?path=` | path | 文件流 | 原始文件(pdf/img/docx/html),静态托管 |
| POST | `/api/knowledge/upload` | multipart + subDir | `{ ok, path }` | 上传文件到 knowledge/ |
| DELETE | `/api/knowledge/doc?path=` | path | `{ ok }` | 删除文档(二次确认在前端) |

**安全**:复用 `fileRoutes.ts` 的 `safeResolve` 路径穿越校验(抽成共享工具 `safePath.ts`)。

**Manifest 数据结构**(TS,放 `shared/src/types/knowledge.ts`):

```ts
export type DocKind = 'md' | 'pdf' | 'docx' | 'html' | 'img';

export interface KnowledgeFileNode {
  type: 'file';
  name: string;        // 原始文件名
  title: string;       // 去前缀去扩展名
  path: string;        // 相对 knowledge/ 的 posix 路径
  kind: DocKind;
  mtime: number;       // Unix 秒
  tags?: string[];
  contentPreview?: string; // 前 2000 字纯文本,供搜索
  links?: string[];    // 出链(解析后的目标 path)
  backlinks?: string[];// 入链(反向索引)
}
export interface KnowledgeDirNode {
  type: 'dir';
  name: string;
  title: string;
  children: KnowledgeNode[];
}
export type KnowledgeNode = KnowledgeFileNode | KnowledgeDirNode;

export interface KnowledgeManifest {
  tree: KnowledgeDirNode;
  flatDocs: KnowledgeFileNode[];
  tags: string[];
  generatedAt: number;
}
```

### 3.2 前端(新增第 4 视图)

```
frontend/src/
├── components/knowledge/
│   ├── KnowledgeView.tsx        # 视图容器(左树 + 右预览 + 顶部搜索/标签)
│   ├── KnowledgeTree.tsx        # 目录树(参考 FileTree)
│   ├── KnowledgeViewer.tsx      # 按 kind 分发预览(md/pdf/img/html/docx)
│   ├── KnowledgeSearch.tsx      # MiniSearch 搜索框 + 结果高亮
│   ├── BacklinksPanel.tsx       # 反向链接面板
│   └── KnowledgeView.css
├── stores/knowledgeStore.ts     # manifest / 当前文档 / 搜索索引 / 标签筛选
└── api/knowledge.ts             # API 封装(参考 files.ts)
```

**改动点**:
- `SidebarNav.tsx`:`View` 类型加 `'knowledge'`,`NAV_ITEMS` 加一项(图标 `BookOpen`)。
- `App.tsx`:加 `currentView === 'knowledge'` 的 `<KnowledgeView />` 挂载块(keep-alive 模式)。

**双向链接渲染**:react-markdown 自定义 remark 插件把 `[[title]]` 转成站内跳转 `<a>`,点击切换 `knowledgeStore.currentPath`。失效链接渲染红色删除线。

**Markdown 复用**:直接用 `MessageContent`(已支持 gfm + 图片相对路径),按需扩展 mermaid/高亮。

### 3.3 文档导入(一次性脚本)

`backend/scripts/import-knowledge.ts`(或 npm script):
1. 读 `D:/Study/KnowledgeBase/docs`。
2. 递归拷贝到 `~/.ai-task-flow/knowledge/`(保留目录结构,过滤白名单扩展名)。
3. 拷贝完触发一次扫描,生成首份 manifest 缓存。
4. 幂等:重复运行覆盖同名文件。

## 五、关键决策与风险

| 决策 | 理由 |
|------|------|
| 不用 PowerShell,改 Node/TS 扫描器 | 跨平台 + 与 Fastify 同栈 + gray-matter 比手写正则健壮 |
| Markdown 复用 MessageContent | 已支持 gfm/图片/引用,避免重复造轮子(符合"组件复用") |
| 文档放数据目录不进 git | 本地优先,内容与代码解耦,不污染仓库 |
| MVP 数据与任务独立 | YAGNI,先交付浏览闭环,打通留接口 |
| 反向链接扫描阶段预建 | 修复原系统缺陷,避免前端运行时 O(n²) |

**风险点**:
- mermaid/高亮在 react-markdown 里的集成成本 → P0 先不做,md 纯文本+代码块可读即可,P1 增量。
- docx 预览 docx-preview 体积较大 → 按需动态 import,不进主 bundle。
- 双向链接 remark 插件有一定复杂度 → P1 再做,P0 先纯渲染。
