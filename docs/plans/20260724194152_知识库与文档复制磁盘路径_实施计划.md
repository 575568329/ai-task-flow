## 一、背景与范围

文档中心(`DocsView` → `TaskDocsView`)与知识库(`KnowledgeView` → `KnowledgeViewer`)共有三类文档,各自定位标识不同:

| 文档类型 | 组件 | 定位标识 | 绝对路径来源 |
|---|---|---|---|
| 知识库文档(md/img/pdf/html/docx) | `KnowledgeViewer.tsx` | `doc.path`(相对 knowledge-base) | **物理根前端拿不到**,需后端在 manifest 暴露 |
| 任务文档 | `TaskDocsView.tsx`(mode='tasks') | `selectedTaskId` | `TaskDTO.taskFilePath`(**后端已返回**,落盘存档) |
| 项目文件 | `TaskDocsView.tsx`(mode='files') | `activeRoot` + `selectedFilePath` | 前端直接拼(root 已是绝对路径) |

**关键结论(已核实源码)**:
- 任务 md **确实落盘**(`<dataDir>/tasks/<taskId>.md`,见 `backend/src/config/dataDir.ts:150-157`),且 `TaskDTO.taskFilePath` 已是绝对路径(`taskRoutes.ts:20-23` 的 `toDTO`),前端 `taskStore` 里每个 task 都带。
- 知识库物理根 `knowledgeDirPath()`(`dataDir.ts:165-172`)只在后端,前端 `getRawUrl` 只是个 HTTP URL,**前端无法自行拼出磁盘路径** → 必须后端暴露。
- 项目文件 `root` 来自系统选择器(`systemApi.selectDirectory`),已是绝对路径;`selectedFilePath` 是相对 root 的 posix 路径,前端拼即可。

### 步骤 5:抽一个共享复制工具(避免三处重复)

文件:`frontend/src/lib/copyPath.ts`(新建)。

```ts
import { toast } from '@/components/ui/toaster';

/**
 * 复制磁盘绝对路径到剪贴板,统一为正斜杠(跨平台、Claude Read / VSCode 友好)。
 * 沿用 shared/utils/prompt.ts 把 taskFilePath 反斜杠转正斜杠的既有约定。
 */
export async function copyDiskPath(raw: string): Promise<void> {
  const normalized = raw.replace(/\\/g, '/').replace(/\/+$/, '');
  try {
    await navigator.clipboard.writeText(normalized);
    toast.success('已复制路径');
  } catch {
    toast.error('复制失败,请手动选择');
  }
}
```

> 两个查看器统一调它,路径归一、toast 一致。三处不重复写 clipboard + toast。

## 三、不该干什么(红线 / 取舍)

1. **不要复制 HTTP URL 或前端深链**。本次需求明确只要「磁盘绝对路径」。`getRawUrl()`(localhost HTTP)、`/#/docs?...`(前端路由)都不在范围 —— 用户已二选一确认只要磁盘路径。
2. **不要为任务文档新增落盘逻辑**。任务 md 已落盘、`taskFilePath` 已在 DTO,直接用,**别再造一套存档或拼路径**(违背 CLAUDE.md「YAGNI / 不重复」)。`taskFilePath` 缺失(历史未派发任务)时按钮置灰即可,不要前端自己猜路径。
3. **不要让前端硬编码 / 猜知识库根**。知识库根必须来自后端 `manifest.basePath`,禁止前端写死 `D:\study\ai-task-flow\knowledge-base` 之类(违背「配置自探测、不依赖外部环境」红线)。
4. **不要改 `buildClaudeArgs` / spawn 逻辑**。那是 TASK-003 步骤 1(Multica 调研)的内容,与本次复制路径无关,别混在一起。
5. **不要扩大化**。本次只加「复制路径」一个按钮,不顺带重构 header、不改导出/下载/编辑逻辑(最小影响面)。
6. **不要在 `scanKnowledge` 里做 IO 之外的事**。只 `basePath: root` 一行,别加时间戳/缓存/日志副作用。
7. **路径分隔符统一正斜杠**,不要按平台输出 `\` —— 与 `shared/utils/prompt.ts` 既有约定一致(Claude Read、VSCode、bash 都接受正斜杠)。

## 五、风险与注意

- **WSL ↔ Windows 路径**:复制出的是 Windows 绝对路径(正斜杠形式),用户若在 WSL 里贴给 Claude,需自行 `/mnt/c/...` 转换。**本次不处理**(超出范围;`get_task` 那套双路径转换是 MCP 层的事,见 CLAUDE.md 红线)。文档/按钮 tooltip 可提示「Windows 路径」。
- **`navigator.clipboard`**:仅 https/localhost 可用。本项目前端固定 localhost:5678,正常;异常时 toast 提示,不静默失败。
- **taskFilePath 缺失**:历史任务(改造前创建、未落盘)可能没有。按钮置灰 + tooltip 说明,不要报错。
- **basePath 随项目迁移变化**:换机器/换目录后 `basePath` 自然变(后端每次扫描取最新 `knowledgeDirPath()`),无需持久化。
