# 前端 shadcn/ui 重构详细计划

## 背景
当前项目混用了自定义 UI 组件和新引入的 shadcn/ui 组件,导致:
1. 文件命名大小写冲突(Windows 不区分,TypeScript 区分)
2. 组件 API 不兼容(旧 Select 用 onChange,新的用 onValueChange)
3. Button variant 不兼容(旧的 "primary",新的 "default")
4. 编译错误 30+ 处

## 目标
统一使用 shadcn/ui 组件库,符合现代 React + TypeScript + Tailwind 设计规范。

```tsx
  </SelectContent>
</Select>
```

### 任务 4.3: 任务抽屉(TaskDrawer)

**目标**: 用 `Sheet` 替换自定义 `Drawer`

**当前实现**:
```tsx
<Drawer open={!!selectedId} onClose={() => setSelectedId(null)}>
  {/* 表单内容 */}
</Drawer>
```

**新实现**:
```tsx
import { Sheet, SheetContent, SheetHeader, SheetTitle, SheetDescription } from '@/components/ui/sheet'

<Sheet open={!!selectedId} onOpenChange={(open) => !open && setSelectedId(null)}>
  <SheetContent side="right" className="w-[600px]">
    <SheetHeader>
      <SheetTitle>{task.title}</SheetTitle>
    </SheetHeader>
    {/* 表单内容 */}
  </SheetContent>
</Sheet>
```

**文件**:

## 阶段 5: 样式规范化

### 任务 5.1: 统一 CSS 变量映射

**问题**: 项目用 `--surface-0` / `--text-1`,shadcn 用 `--background` / `--foreground`

**解决方案**: 在 Tailwind 配置里映射
```js
// tailwind.config.js
theme: {
  extend: {
    colors: {
      background: 'var(--surface-0)',
      foreground: 'var(--text-1)',
      border: 'var(--border-primary)',
      // ... 映射所有变量
    }
  }
}
```

---

### 任务 5.2: 移除硬编码 style

**规范**: 优先使用 Tailwind class,避免 inline `style={{}}`

**示例**:
```tsx
// 旧
<div style={{ padding: 16, color: 'var(--text-2)' }}>

// 新
<div className="p-4 text-muted-foreground">
```

## 附录 B: 常见错误和解决方案

### 错误 1: `Property 'inline' does not exist`
**文件**: `MessageContent.tsx`
**原因**: react-markdown 的 `code` 组件 props 类型不匹配
**解决**: 
```tsx
code: ({ node, inline, className, children, ...props }: any) => {
  // 暂时用 any 绕过类型检查
}
```

### 错误 2: `File name differs in casing`
**原因**: Windows 不区分大小写,Git 里同时存在 `Button.tsx` 和 `button.tsx`
**解决**: 删除所有大写开头的旧组件文件

### 错误 3: `Type 'primary' is not assignable`
**原因**: shadcn Button 的 variant 只有 default/destructive/outline/secondary/ghost/link
**解决**: 用 `button-compat.tsx` 映射,或手动改成 `variant="default"`
