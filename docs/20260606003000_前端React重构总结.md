# 前端 React + Tailwind 重构总结

> 完成日期:2026-06-05
> 范围:前端从 Vue 3 重构为 React 18 + Tailwind,补全缺口 + 小改后端配合
> 实施计划见 [`docs/plans/20260605224500_前端React重构实施计划.md`](plans/20260605224500_前端React重构实施计划.md)

---

## 一、重构动机

代码审查 + 联调中发现现有 Vue 前端存在三类问题:

1. **功能缺口(commit 信息夸大)**:拖拽改状态、Diff 审查在 commit 里声称已实现,实际代码中**从未实装**。
2. **架构债**:前后端类型各维护一份(漂移风险);审查相关领域事件(approve/reject)被 MVP 简化掉,不发事件,SSE 无法联动。
3. **体积**:Element Plus 全量打包,bundle 达 1.2MB。

用户要求改用 React + Tailwind 重写,借机一并补齐。

## 二、技术栈变更

| 关注点 | 重构前 | 重构后 |
|--------|--------|--------|
| 框架 | Vue 3 | React 18 |
| 样式 | Element Plus(组件库) | Tailwind CSS v4 + 自建组件 |
| 状态 | Pinia | Zustand |
| 拖拽 | 无 | @dnd-kit |
| Diff | diff2html(未接入) | react-diff-view |
| 图标 | @element-plus/icons | lucide-react |
| HTTP | axios | 原生 fetch 封装 |
| 类型 | 前端自维护一份 | `@ai-task-flow/shared` 共享包 |

- **F9**:项目过滤 + 标题/ID 搜索(纯前端 selector)。

### 阶段四 · 收尾

## 六、关键设计决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 状态管理 | Zustand 包干,不引 TanStack Query | 看板规模小,依赖更少 |
| 组件库 | 自建 Tailwind 组件,不引 Radix/shadcn | 体积可控,无额外依赖 |
| 主题 | CSS 变量 + `.dark` 类策略 | token 集中,组件零改动适配明暗 |
| SSE 合并 | 收事件后按 aggregateId 重新拉取该任务 | 事件只带状态变化,拉取保证数据完整 |
| Vue 旧码 | 直接删除(git 可找回) | 避免仓库双份前端 |

## 七、遗留事项 / 未来优化

- **Diff baseBranch**:前端 DiffViewer 默认走后端 `main`,若仓库默认分支是 `master` 需传 `?base=`;可优化为后端自动探测默认分支。
- **审查事件 merge 动作**:approve 目前只改状态发事件,未真正执行 git merge(MVP 简化,worktree 仍需手动合并)。
- **多 Agent / AI 任务拆解**:竞品对比中的两个短板仍在(见 `docs/20260605223500_竞品对比分析.md`),属下一阶段。
- **H-1 安全项**:HTTP 默认监听 `0.0.0.0` 且无鉴权,本次未动(需用户决定),见代码审查报告。
- **bundle 进一步优化**:react-diff-view 可改懒加载(仅 review 抽屉用到)。
