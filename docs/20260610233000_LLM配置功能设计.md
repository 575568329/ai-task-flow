# LLM API 配置功能设计文档

> 版本：v1.0 | 日期：2026-06-10 | 状态：设计中

## 1. 背景与目标

### 1.1 当前问题

调研资料功能（Chat）依赖 LLM 进行聊天和资料检索，但当前 LLM 的 API 地址、API Key、模型名**只能通过环境变量配置**：

```typescript
// backend/src/http-server.ts
const llmProvider = new OpenAiCompatibleProvider(
  process.env.LLM_BASE_URL || 'https://open.bigmodel.cn/api/paas/v4',
  process.env.LLM_API_KEY || '',
  process.env.LLM_MODEL || 'glm-4-plus',
);
```

用户无法在 Web 界面修改这些配置，每次切换 LLM 服务商（如智谱 → OpenAI → DeepSeek）都需要修改环境变量并重启服务。

### 1.2 目标

- 在前端提供 LLM 配置界面，支持修改 **API 地址、API Key、模型名**
- 配置保存到后端 JSON 文件，**立即生效**（无需重启）
- API Key 脱敏显示，前端不传输明文

## 2. 配置优先级

```
llm-config.json 文件  >  环境变量 (LLM_BASE_URL / LLM_API_KEY / LLM_MODEL)  >  硬编码默认值
```

## 3. 架构设计

### 3.1 数据流

```
[前端 Settings Modal]
    │
    │  GET /api/llm-config (读取，apiKey 脱敏)
    │  PUT /api/llm-config (保存)
    ▼
[llmConfigRoutes.ts]
    │
    ▼
[LlmConfigService] ──── 应用层：配置读写 + 动态热更新
    │
    ▼
[JsonLlmConfigRepository] ──── 基础设施层：JSON 文件读写
    │
    ▼
[~/.ai-task-flow/llm-config.json]
    │
    │  配置变更后重建
    ▼
[OpenAiCompatibleProvider] ──── 动态替换实例
    │
    ▼
[ChatService] ──── 通过 getProvider() 获取最新实例
```

### 3.2 DDD 分层映射

| 层级 | 文件 | 职责 |
|------|------|------|
| **Domain** | `LlmConfigEntity.ts` | 值对象定义 + maskApiKey 工具函数 |
| **Domain** | `LlmConfigRepository.ts` | 仓储接口 |
| **Infrastructure** | `JsonLlmConfigRepository.ts` | JSON 文件持久化 |
| **Application** | `LlmConfigService.ts` | 配置管理 + Provider 热更新 |
| **Interface** | `llmConfigRoutes.ts` | REST API |

## 4. API 接口设计

### 4.1 GET /api/llm-config

获取当前 LLM 配置（API Key 脱敏）。

**响应 `200 OK`：**

```json
{
  "baseURL": "https://open.bigmodel.cn/api/paas/v4",
  "apiKeyMasked": "sk-1********4a",
  "apiKeySet": true,
  "model": "glm-4-plus"
}
```

**无配置时**（返回环境变量 / 默认值）：

```json
{
  "baseURL": "https://open.bigmodel.cn/api/paas/v4",
  "apiKeyMasked": "",
  "apiKeySet": false,
  "model": "glm-4-plus"
}
```

### 4.2 PUT /api/llm-config

保存 LLM 配置，立即生效。

**请求体：**

```json
{
  "baseURL": "https://api.openai.com/v1",
  "apiKey": "sk-new-key-here",
  "model": "gpt-4o"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `baseURL` | string | 是 | 必须是合法 URL |
| `apiKey` | string | 否 | 空字符串 = 保持原值不变 |
| `model` | string | 是 | 非空字符串 |

**校验失败 `400`：**

```json
{ "error": "baseURL 格式无效" }
```

## 5. 前端交互设计

### 5.1 入口位置

全局导航栏（AppNav）右侧添加 **齿轮图标**，点击弹出设置 Modal。

### 5.2 交互流程

```
用户点击齿轮图标
  → GET /api/llm-config 加载当前配置
  → 打开 Modal，回填 baseURL + model
  → apiKey 字段为空，placeholder 提示"留空保持原值"
  → 用户修改字段后点击"保存"
  → PUT /api/llm-config
  → 成功：toast "配置已保存并生效"，关闭 Modal
  → 失败：toast 错误信息，Modal 保持打开
```

### 5.3 设计原则

- **操作路径最短化**：点击齿轮 → 直接弹出 Modal，一步到位
- **安全感**：API Key 使用 `type="password"`，已设置的 Key 显示脱敏值
- **即时反馈**：保存成功后 toast 提示，新配置立即生效
- **零配置可用**：不设置也能使用，走环境变量默认值

## 7. 安全注意事项

1. **API Key 脱敏**：后端返回给前端时只显示 `sk-1********4a` 格式
2. **前端不回填 Key**：编辑时 apiKey 字段为空，用户不修改则发空字符串
3. **文件权限**：`llm-config.json` 包含明文 API Key，保存时设置 `chmod 0600`
