# RPP-Web 前端技术复盘

**项目**: rpp-web (资源加工平台前端)  
**技术栈**: Vue 2.6 + Vuex 3 + Vue Router 3 + Webpack 3  
**业务**: 众包加工 + 教辅加工（30+ 种加工类型）

---

## 一、核心业务架构

### 1.1 动态任务处理系统 ⭐⭐⭐⭐⭐

**问题**: 如何用一套架构支撑 30+ 种不同的加工类型？

**方案**: Vue 动态组件 + 策略映射

```vue
<!-- src/views/common/task-process/index.vue -->
<component :is="currentComponent" :task="taskData"></component>
```

```javascript
// 根据 processTab.value (数值编码) 映射到具体组件
const componentMap = {
  1: 'QuestionResourceMarkProcess',    // 试题标注
  2: 'QuestionResourceAuditProcess',   // 试题审核
  13: 'DivideTopicProcess',             // 划题
  25: 'FuseEnteringProcess',            // 融合资源录入
  44: 'QuestionResourceAdaptProcess',   // 试题改编
  // ... 共 30+ 种
};
```

**编码定义**: `src/enum/rpp/workShopConstant.js` → `ProcessTypeKeyEnum`

**价值**:
- ✅ 避免为每种加工类型创建独立路由
- ✅ 统一的加工流程控制（领任务 → 加工 → 提交 → 判定）
- ✅ 组件按需加载，减少初始包体积

---

### 1.2 API 层架构

**设计**: 服务对象模式 + 统一拦截器

```javascript
// src/api/pkgService.js
import { generateUrl, get, post } from '../utils/http';

export default {
  getPkgDetail(pkgId) {
    return get(generateUrl(`api/pkg/getPkgDetail`), { pkgId });
  }
};
```

**请求拦截器** (`src/utils/http.js`):
- 自动注入鉴权头: `Authorization: ZHIXUE`
- 多语言头: `locale` (页面语言) + `localeWorkShop` (工作室语言)
- 自定义鉴权支持（公共试题编辑器）

**响应拦截器**:
- 会话失效检测: HTTP 999 → 跳转登录
- 权限错误统一处理: 900/901/902 状态码
- 错误信息国际化

---

### 1.3 路由架构

**组织**: 按业务域拆分 12+ 个模块

```
src/router/
├── common/          # 通用（下载中心、任务加工、任务结果）
├── rpp/             # 工厂主业务（订单、工作室、我的任务）
├── redesign/        # 新版平台
├── xgk/             # 新高考

├── crowdsourced/    # 众包管理
├── assistant/       # 小助手
└── toolbox/         # 资源工具箱
```

**路由守卫** (`src/router/router-handle.js`):
- 老路由 → 新路由自动重定向
- 平台配置驱动的功能开关
- 自定义鉴权参数初始化
- 大数据埋点配置

### 2.2 国际化系统 ⭐⭐⭐⭐⭐

**问题**: 如何支持四语言 + 工作室/页面双语言切换？

**支持语言**:
- `zh-CN` - 简体中文
- `zh-TW` - 繁体中文（澳门项目）
- `en-US` - 英文
- `pt-BR` - 葡萄牙语

**架构** (2026-06 重构):

```
src/config/lang/locales/
├── zh-CN/
│   ├── workShop.json      # 工作室核心
│   ├── taskProcess.json   # 任务加工
│   ├── taskHall.json      # 众包大厅
│   ├── myTask.json        # 我的任务
│   ├── common.json        # 全局词典
│   └── _rest.json         # 冷门 namespace（平铺合并）
├── zh-TW/   # 同上
├── en-US/   # 同上
└── pt-BR/   # 同上
```

**语言切换逻辑**:
```javascript
// 工作室路由：优先工作室语言，无工作室语言时回退 zh-CN
// 非工作室路由：使用页面语言
i18n.t = function(key, localeOrValues, values) {
  const locale = resolveBizLocale();  // 根据路由自动判断
  return rawTranslate(key, locale, localeOrValues);
};
```

**质量保障**:
- ✅ Husky pre-commit 检查: `npm run i18n:check`
- ✅ 路径错位检测（key 所在目录与 namespace 不匹配）
- ✅ 漏翻检测（四语言未对齐）
- ✅ VSCode 插件 `i18n-ally` 实时显示

**相关文档**: `docs/6月版本开发/多语言改造规范手册.md` (47 个历史 bug 警示)

## 三、性能优化实践

### 3.1 构建优化

**内存管理**:
```json
"dev": "NODE_OPTIONS=--max_old_space_size=8192 webpack-dev-server",
"test": "NODE_OPTIONS=--max_old_space_size=16384 node build/build.js"
```

**为什么需要这么大内存？**
- Webpack 3 构建效率低
- 依赖包多（80+ 个）
- 大量全局 CSS 导入

**SVG 优化**:
```javascript
// webpack.base.conf.js - SVG 雪碧图
{
  test: /\.svg$/,
  include: [resolve('src/assets/icons/svg')],
  loader: 'svg-sprite-loader',
  options: { symbolId: 'icon-[name]' }
}
```

**图片 Base64**:
- PNG: limit 400KB
- GIF: limit 300KB  
- JPG: limit 10KB

**效果**: 减少小图标 HTTP 请求，提升首屏加载速度

## 六、核心业务流程

### 6.1 任务加工流程

```
1. 用户登录 → SSO/域账号鉴权
2. 选择角色 → 工厂/众包
3. 进入工作台/任务大厅
4. 领取任务 → API 获取任务包
5. 跳转加工页 → /common/task-process?processType=1&taskId=xxx
6. 动态加载组件 → 根据 processType 映射
7. 加工操作 → 标注/划题/校对
8. 提交审核 → API 提交，更新状态
9. 判定结果 → 合格/不合格/申诉
```

---

### 6.2 工作室管理流程

```
1. 创建工作室 → 配置人员/任务类型/绩效规则
2. 创建订单 → 分配任务包
3. 实时监控 → 进度/质量/人员
4. 结算薪资
```
