## 3. 💡 关键技术决策

| 决策点 | 备选方案 | 最终选择 | 理由 |
|--------|---------|---------|------|
| 翻译出口 | $t / $tm / studioI18nT 混用 | Vue 组件 `$tm()`、JS `i18n.t()`，禁 `$t()` | 出口收敛，卡口好扫 |
| 语言判断 | 全局单一语言 | 按路由：工作室路由用工作室语言、非工作室用页面语言 | 澳门工作室与大陆隔离 |
| 缺失回退 | 报错/空 | 工作室路由 + languageLabel 空 → 固定回退 zh-CN | 防大陆工作室错显多语言 |
| 卡口策略 | 全量扫裸中文 | 基线增量，只校验 key 存在+四语言齐全，count=0 | 存量太多会卡死全员提交 |
| namespace | 每模块独立 JSON | >100key 或引用>50次才独立，否则 _rest.json 平铺 | 避免碎文件 |
| 冒号格式 | 半角 / 半角+空格 / 全角 | 全角冒号 `：` | HTML 压缩连续空格，半角+空格无效 |
| 品牌名 | 翻译 | 固定中文硬编码 | 业务要求品牌名不随语言变 |
| 列宽语言源 | getCurrentLocale | resolveBizLocale/getStudioLocale | 让容器宽度语言与 `$tm` 渲染语言同源 |
| 表头省略 | 每列 slot / 全局 directive / mixin+render-header | mixin + `:render-header` | 改动小，不与已有 slot="header" 冲突 |
| 模块级 i18n 冻结 | computed | methods 返回 | Vue2 初始化序 methods→data→computed，data() 拿不到 computed |
| 枚举国际化 | name 存中文 | name 存 i18n key，判断用 code | 调用方套 `$tm`，判断不受语言影响 |
| 跨应用传语言 | 维护 Vue↔Angular | 不维护，全撤回 | 一度实现双向 `?lang=`，后决定不维护其他应用 |
| 整页漏翻检测 | 塞进卡口扫中文 | 撤出卡口，靠枚举清单 | 卡口只管机械正确性，覆盖率是范围/计划问题 |

## 4. 🐛 踩坑记录

### 坑：cut-and-transfer 判断文案多语言化失效（核心）
- **现象**：英/葡工作室答案为空时，不弹"添加答案"确认框直接提交。
- **根因**：`res.content`（后端返回）`=== $tm('divideTopic.noAnswerYet')`，但 en/pt 译文（`No answer yet`/`Sem resposta ainda`）与后端返回值（`No answer available`/`Sem resposta disponível`）不一致 → 切语言判断恒 false。简繁一致所以不暴露。
- **解决**：改 en-US/pt-BR 的 `noAnswerYet` 值对齐后端。
- **预防**：翻译文案绝不能当判断条件；卡口只查 key 存在，查不出"值与后端不一致"。

### 坑：加工页整页无限 loading（核心真业务 bug）
- **现象**：标注基础题页面一直 loading。
- **根因**：`allXKnowledges` 为 null 时，`findRootNode` 的 `for...of` 抛 `not iterable`，异常被 `.catch` 吞掉且不再 `$emit('initComplete')`。深层：流程配了"新课标解题方法"却没配"新课标知识点"（配置组合 bug）+ 代码健壮性缺陷。
- **解决**：`handleFactorySolveMethodZF` 加空值保护。
- **预防**：树遍历工具（`for...of`）必须加空值保护；async 链路 `.catch` 吞异常会造成静默卡死。

### 坑：前人改造遗留的编译/渲染错误
- `:v-text=` 加冒号致文案不渲染（11处，SingleLinkManageCard）
- 全角引号 `"` 致编译失败（Vue 4处 / Angular login.component.ts **120个错误**）
- 前人误删 4 个 data() 列表（从 git 恢复改 computed）
- JSON key 与代码引用不一致（19个，前人预留 52key 命名不符）
- `$tm` 漏 `:` 绑定前缀致页面显示 key 原文（13处，PersonalInfoManageModule）

### 坑：i18n 机制类
- 双冒号（key 自带冒号+模板拼`：`，全项目仅 List.vue 2处）
- constant.js 翻译函数 this 丢失（`$tm` 方法引用传 JS 丢 this，改 `i18n.t`）
- 占位符四语言不一致（`{type}` 在 en/pt 缺失，传参被丢）
- 葡文单复数语法陷阱（`{type} Disponíveis` 性数不匹配，改动词不定式绕开）
- JS 里误用 `{{ $tm() }}` 模板语法致编译失败
- 模块级 `i18n.t()` 冻结（import 期求值锁语言，靠 reload 兜底）
- custom-uploader 误改乌龙（`errorType===$tm('fileExitsList')` 改 `$tm` 后澳门非简体恒不等；查证 errorType 来自 webuploader SDK 硬编码中文，4 兄弟组件都保留中文比较、仅此一个被误改）
- VerifyDialog `new Vue` 未注入 i18n 致 `$tm` 崩
- 单选题 answer=D/answerJson=B 脏数据（answerJson 在 noTransJsonSectionCode 白名单不回写，读脏旧值）

### 坑：工具/环境
- lint-staged Windows 命令行超长（272 文件拼 18864 字符超限，`--no-verify` 跳过）
- i18n baseline 记忆过时（记忆说 47 个 bug 冻结，实际 count=0）
- `?lang=` query 位置（Angular 只解析 hash 后 query）

## 5. 📚 经验与收获

详见 [开发经验_rpp-web国际化.md](开发经验_rpp-web国际化.md)。核心几条：

1. **卡口管正确性、覆盖率靠枚举清单**——静态卡口回答"代码对不对"，回答不了"该写的都写了没"。
2. **AI 批量改造遗漏是结构性盲区**——第一步必须先输出文件清单经确认，禁止边发现边改。
3. **翻译文案当判断条件是高危反模式**——判断用 code/英文常量，`$tm` 只负责显示。
4. **翻译长度是布局主要风险源**——葡文常比中文长 2~4 倍。
5. **正则做跨变量数据流追踪本质脆弱**——命中数剧烈震荡（34→14→1→72）是工具触及天花板的信号。
