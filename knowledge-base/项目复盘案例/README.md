## 📁 案例清单

### 1. rpp-web 澳门国际化二期

**项目背景**:Vue2 资源加工平台前端的多语言改造二期(zh-CN/zh-TW/en-US/pt-BR 四语言)。

**产出文件**:
- [复盘文档](20260701_澳门国际化二期_复盘.md) — 记录"做了什么":任务清单、关键决策、踩坑、问题
- [开发经验](开发经验_rpp-web国际化.md) — 提炼"下次怎么做更好":可复用技术方案、避坑清单

**复盘特点**:
- 经"归纳者→独立审阅者"两轮制衡(分角色机制)
- 16 个 commit hash 全部核实真实存在
- 关键数量(key 数、文件数)对照原始会话文本验证

**技术亮点**:
- 静态 i18n 卡口(基线增量策略)
- 语言决策集中到单一汇聚点(`resolveBizLocale`)
- 翻译长度导致的布局适配(葡文比中文长 2~4 倍)
- BARE_RENDER 检测(抓"key 存进 JS 常量、模板漏包 `$tm`")

**适用参考场景**:
- 存量 Vue 项目引入 i18n
- 多租户/多区域语言策略
- 中文→英/葡的 UI 布局适配
- i18n 卡口设计(渐进收紧,不阻塞历史遗留)

## 🔗 关联

**生成工具**:这些复盘文档由 [`project-retrospective` skill](../技术方案/20260701_项目复盘方案总纲.md) 产出。

**方法论**:
- 脚本过滤噪音(discover_sessions.ps1 + extract_clean.ps1)
- Claude 分角色分析(归纳者 + 独立审阅者)
- 最多三轮交人审批(防无限循环)
- 三路产出:复盘文档 + 记忆沉淀 + 开发经验

**如何复用**:
1. 技术方案可直接参考(如 i18n 卡口设计、语言决策汇聚)
2. 避坑清单可纳入 checklist(如"翻译文案不做逻辑判断")
3. 复盘结构可作为自己项目复盘的模板参考
