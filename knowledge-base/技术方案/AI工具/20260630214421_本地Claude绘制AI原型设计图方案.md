# 用本地 Claude 进行 AI 原型/设计图绘制 — 方案调研

> 调研日期:2026-06-30
> 核心问题:能否用本地的 Claude(Claude Code)来绘制原型或设计图,类似蓝湖、MasterGo 那种,并能切片导出图片素材用于网页开发。
> 结论先行:**可以。当前生态以 Figma 为事实标准,推荐 Claude Code + Figma MCP 的 `use_figma` 工具,直接在 Figma 画布生成可编辑、可切片的设计图。**

---

## 一、先厘清:两种不同的"产物形态"

选错方向会走弯路,先分清你要的是哪一种:

| 产物形态 | 是什么 | 谁擅长 |
|---------|--------|--------|
| **可编辑的设计稿**(矢量图层、画板、组件,可在画布拖拽改、可切片导出图片) | Figma/MasterGo 那种文件 | 设计工具 + MCP |
| **HTML/代码原型**(浏览器能跑、能点的高保真页面) | 代码即设计 | Claude 本身最强 |

本需求(**生成完整 UI 图 → 在上面切片 → 拿图片物料做网页**)属于第一种:**可编辑、可切片的设计稿**。

---

## 二、最终推荐方案

### 方案 A:Claude Code + Figma MCP 的 `use_figma` 工具 ⭐ 首选

这是"**本地 Claude 直接生成设计图**"的正向路径,不经过代码中间层。

**能力**(来自 Figma 官方教程,2026-05 推出的 `use_figma` 工具):
- 直接在 Figma 画布上创建 / 编辑 Frame、矩形、文本、组件
- 自动应用 Auto Layout
- 调用你 Figma 里已有的 Design System(组件库 / 变量 / 样式)生成符合规范的设计
- 输出可编辑图层,可直接在 Figma 切片导出图片素材

**工作流:**
```
终端对 Claude 说:"用 use_figma 和我的设计系统,为搜索页生成一个空状态"
     ↓
Claude 通过 MCP 调用 use_figma 工具
     ↓
直接在 Figma 画布生成可编辑 Frame(使用你的真实组件)
     ↓
在 Figma 里切片导出图片素材 → 用于网页开发
```

**限制:**
- 需要你 Figma 里**已有设计系统**(组件 / 变量),才能生成符合规范的设计;Figma 是空的时只能用基础形状拼
- 截至 2026-06 仍在 Beta,Figma 称未来会变成按量付费功能,当前免费

### 方案 B:Figma Make(Figma 自带 AI,零配置)⭐ 备选

- **不依赖本地 Claude**,是 Figma 云端自己的 AI
- 文本提示 → 直接生成可编辑的 Figma 矢量设计稿(不是代码)
- 可上传参考图、调颜色字体,开箱即用、质量稳定
- 若只看重"快速拿到可切片设计图"这个结果,这条路最省事

### 方案 C:Figma Agent(2026-05 推出,更高级)

- 比 Make 更强的设计 Agent,接受复杂任务、自动调用组件库与 Design Tokens
- 可接入自定义工具 / 社区技能
- 同样在 Figma 内,不在本地终端

## 三、Figma MCP 配置步骤(方案 A 落地)

> 前置:需要 Figma 的 **Dev 或 Full 付费席位**;安装 Figma 桌面版;本地已装 Claude Code。

1. **安装官方 Figma 插件**(自带 MCP 配置 + Agent Skills):
   ```bash
   claude plugin install figma@claude-plugins-official
   ```

2. **在 Figma 开启 MCP server**(二选一):
   - 远程服务器(推荐,大多数人用这个):OAuth 登录即可,功能最全
   - 桌面服务器:Figma 桌面版 → Preferences → 开启 Dev Mode MCP Server 开关,本地起服务(仅 Figma 打开时运行)

3. **手动配置(若不用插件):**
   ```bash
   # 远程
   claude mcp add --scope user --transport http figma <Figma 给的 URL>
   # 或桌面版
   claude mcp add --transport sse figma-dev-mode-mcp-server <本地URL>
   ```
   用 `claude mcp list` 验证连接状态。

4. **使用**:对话里给 Claude 你的 Figma 文件链接 + 提示词,例如:
   > "用 use_figma 和我的设计系统,为这个登录页面 [Figma链接] 生成一个暗色主题变体"

### Figma MCP 三个核心工具

| 工具 | 方向 | 用途 |
|------|------|------|
| `use_figma` | 正向生成 | **直接在画布创建/编辑设计**(本需求主力) |
| `generate_figma_design` | 代码 → 设计 | 把渲染后的 HTML/React 页面转成 Figma 可编辑图层(Claude Code 独占 + 仅远程 MCP) |
| `get_design_context` / `get_code` | 设计 → 代码 | 读 Figma 设计稿生成代码 |

## 四、备选方案全景对比

| 方案 | 本地 Claude 主导 | 产物 | 成本 | 适合场景 |
|------|:---:|------|------|---------|
| **Claude Code + Figma MCP** | ✅ | 可编辑设计稿 | Figma 付费席位 + Token | 官方加持、能力最全、双向打通 |
| **Figma Make / Agent** | ❌ | 可编辑设计稿 | Figma AI Credits | 零配置、质量稳定、最省事 |
| **Penpot + MCP** | ✅ | 可编辑设计稿 | 免费/自托管 | 要免费开源、数据本地、能折腾 |
| **MasterGo / 蓝湖 MCP** | 部分 | 主打设计转代码 | 国内云付费 | 国内团队、已有设计稿转代码 |
| **本地 Claude 直接出 HTML** | ✅ | 代码原型(不可切片) | 仅 Token | 只要能跑的高保真原型 |
| **Claude Design** | ❌(云端) | 代码型原型 | Anthropic 付费 | 快速验证想法 |

### 各备选要点

- **Penpot**:开源 Figma 替代,可 Docker 自托管,官方 MCP 已集成进主仓库(2026-02 起),MCP 对所有人免费。社区实测:配 Claude Opus 效果惊艳,但配本地小模型不稳定且 Token 消耗巨大。
- **MasterGo / 蓝湖**:均有官方 MCP,但方向是"读已有设计稿 → 生成代码"(D2C),不是"从零画设计稿";AI 能力多绑定其自家云平台。
- **本地 Claude 直接出 HTML**:配合设计规范 skill(如 Spark-design)可出高保真可点原型,无需任何设计工具——但产物是代码,**不能切片导出图片素材**,不满足本需求。

## 五、生态结论

**AI 设计生态当前(2026-06)高度向 Figma 集中,Figma 是事实标准。** 证据:

1. **MCP 客户端目录**:Claude Code、Cursor、VS Code、Codex、Gemini CLI、Kiro、Copilot 等几乎所有主流 AI 编码工具接的都是 Figma MCP。
2. **Anthropic 自己**:官方插件市场首推的设计插件就是 Figma 的,没强推自家 Claude Design 绑定 Claude Code。
3. **双向打通**:正向画(`use_figma`)、代码转设计(`generate_figma_design`)、设计转代码(`get_code`)读写全通,目前只有 Figma 做到这个完整度。
4. **战略选择**:Figma 开放了 MCP 写画布权限给所有第三方 AI 工具,走开放协议而非围墙花园,因此吸附了最完整的工具生态。

**跳出 Figma 的唯二理由**:
- 强数据本地化诉求 → Penpot 自托管
- 纯粹只要代码原型,不要可切片设计稿 → 本地 Claude 直接写 HTML

**针对本需求(生成可切片 UI 设计图 + 愿意为 Figma 付费)的最终判断:Figma + `use_figma` MCP 是当前成熟度和性价比最高的组合。**
