## 一、性能指标(Core Web Vitals)

### 1. 关键指标

| 指标 | 含义 | 目标 | 影响 |
|------|------|------|------|
| **LCP** (Largest Contentful Paint) | 最大内容绘制 | < 2.5s | 首屏体验 |
| **FID** (First Input Delay) | 首次输入延迟 | < 100ms | 交互响应 |
| **CLS** (Cumulative Layout Shift) | 累积布局偏移 | < 0.1 | 视觉稳定性 |
| **TTFB** (Time to First Byte) | 首字节时间 | < 600ms | 服务端性能 |
| **FCP** (First Contentful Paint) | 首次内容绘制 | < 1.8s | 感知速度 |
| **TTI** (Time to Interactive) | 可交互时间 | < 3.8s | 可用性 |

**监控代码**:

```js
// Performance Observer
const observer = new PerformanceObserver(list => {
  list.getEntries().forEach(entry => {
    if (entry.name === 'largest-contentful-paint') {
      console.log('LCP:', entry.renderTime || entry.loadTime)
    }
  })
})
observer.observe({ entryTypes: ['largest-contentful-paint'] })

// Web Vitals 库(推荐)
import { getLCP, getFID, getCLS } from 'web-vitals'

getLCP(console.log)
getFID(console.log)
getCLS(console.log)
```


### 3. 通用优化

**防抖/节流**:

```js
// 防抖(延迟执行,重复触发重新计时)
function debounce(fn, delay) {
  let timer = null
  return function(...args) {
    clearTimeout(timer)
    timer = setTimeout(() => fn.apply(this, args), delay)
  }
}

// 节流(固定频率执行)
function throttle(fn, delay) {
  let last = 0
  return function(...args) {
    const now = Date.now()
    if (now - last > delay) {
      fn.apply(this, args)
      last = now
    }
  }
}

// 使用
input.addEventListener('input', debounce(search, 300))
window.addEventListener('scroll', throttle(handleScroll, 200))
```

**requestAnimationFrame**(动画):

```js
// ❌ setInterval(卡顿)
setInterval(() => {
  element.style.left = left++ + 'px'
}, 16)

// ✅ requestAnimationFrame(丝滑)
function animate() {
  element.style.left = left++ + 'px'
  requestAnimationFrame(animate)
}
requestAnimationFrame(animate)
```

**IntersectionObserver**(懒加载/曝光埋点):

```js
const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target
      img.src = img.dataset.src  // 加载真实图片
      observer.unobserve(img)
    }
  })
})

document.querySelectorAll('img[data-src]').forEach(img => {
  observer.observe(img)
})
```

## 四、后端优化

### 1. 数据库优化

**索引**:

```sql
-- 单列索引
CREATE INDEX idx_user_email ON users(email);

-- 复合索引(最左匹配)
CREATE INDEX idx_user_status_created ON users(status, created_at);

-- 查看索引使用情况
EXPLAIN SELECT * FROM users WHERE email = 'test@example.com';
```

**慢查询优化**:

```sql
-- 避免 SELECT *
SELECT id, name FROM users;  -- ✅

-- 分页优化(游标)
SELECT * FROM posts WHERE id > last_id ORDER BY id LIMIT 20;  -- ✅ 游标
SELECT * FROM posts LIMIT 10000, 20;  -- ❌ OFFSET 慢
```

**连接池**:

```js
// Node.js (pg)
const { Pool } = require('pg')
const pool = new Pool({
  max: 20,  // 最大连接数
  idleTimeoutMillis: 30000,
  connectionTimeoutMillis: 2000
})

const result = await pool.query('SELECT * FROM users')
```

**读写分离**:

```
写请求 → 主库(Master)
         ↓ 同步
读请求 → 从库(Slave 1/2/3)
```

### 2. 缓存策略

**多级缓存**:

```
浏览器缓存(Memory/Disk)
   ↓ Miss
CDN 缓存
   ↓ Miss
Nginx 缓存(代理缓存)
   ↓ Miss
应用缓存(Redis/Memcached)
   ↓ Miss
数据库
```

**Redis 缓存**:

```js
const redis = require('redis')
const client = redis.createClient()

async function getUser(id) {
  // 1. 查缓存
  const cached = await client.get(`user:${id}`)
  if (cached) return JSON.parse(cached)
  
  // 2. 查数据库
  const user = await db.users.findById(id)
  
  // 3. 写缓存(TTL 1小时)
  await client.setEx(`user:${id}`, 3600, JSON.stringify(user))
  
  return user
}
```

**缓存穿透**(查不存在的数据):

```js
// 解法1:缓存空值
if (!user) {
  await client.setEx(`user:${id}`, 60, 'null')
}

// 解法2:布隆过滤器
if (!bloomFilter.has(id)) {
  return null  // 一定不存在
}
```

**缓存雪崩**(大量缓存同时失效):

```js
// 解法:随机 TTL
const ttl = 3600 + Math.random() * 600  // 3600~4200秒
await client.setEx(key, ttl, value)
```

**缓存击穿**(热点 key 失效):

```js
// 解法:互斥锁
const lock = await redisLock.acquire(`lock:${id}`)
try {
  const cached = await client.get(`user:${id}`)
  if (cached) return cached
  
  const user = await db.users.findById(id)
  await client.setEx(`user:${id}`, 3600, JSON.stringify(user))
  return user
} finally {
  await lock.release()
}
```

## 七、案例分析

### 案例1:首屏优化(电商首页)

**优化前**:LCP 5.2s

**优化措施**:
1. Hero 图片换 WebP,预加载(LCP -1.5s)
2. 首屏 CSS 内联,JS `defer`(FCP -0.8s)
3. 商品列表 SSR(TTFB -0.6s)
4. CDN 加速(TTFB -0.3s)

**优化后**:LCP 2.0s ✅

### 案例2:长列表优化(消息列表)

**优化前**:10000条消息,滚动卡顿

**优化措施**:
1. 虚拟滚动(react-window),只渲染可视区
2. 图片懒加载(IntersectionObserver)
3. 消息内容防抖更新

**优化后**:丝滑60fps ✅
