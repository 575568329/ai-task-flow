## 一、模块化演进

### 1. 模块化方案

| 方案 | 环境 | 特点 |
|------|------|------|
| **IIFE** | 浏览器 | 立即执行函数,作用域隔离 |
| **CommonJS** | Node.js | 同步加载,`require/module.exports` |
| **AMD** | 浏览器 | 异步加载,RequireJS |
| **UMD** | 通用 | 兼容 CommonJS/AMD/全局变量 |
| **ESM** | 现代 | 静态分析,`import/export`,Tree-shaking |

**ESM vs CommonJS**:

```js
// CommonJS(运行时)
const fs = require('fs')  // 动态,运行时加载
module.exports = { foo }

// ESM(编译时)
import fs from 'fs'  // 静态,编译时分析
export { foo }
```

## 二、Webpack

### 1. 核心概念

**Entry(入口)**:

```js
module.exports = {
  entry: './src/index.js',  // 单入口
  // 或
  entry: {
    app: './src/app.js',
    admin: './src/admin.js'  // 多入口
  }
}
```

**Output(输出)**:

```js
output: {
  path: path.resolve(__dirname, 'dist'),
  filename: '[name].[contenthash].js',  // 文件名(含hash)
  clean: true  // 构建前清空
}
```

**Loader(转换器)**:

```js
module: {
  rules: [
    {
      test: /\.css$/,
      use: ['style-loader', 'css-loader']  // 从右往左执行
    },
    {
      test: /\.tsx?$/,
      use: 'ts-loader'
    },
    {
      test: /\.(png|jpg|gif)$/,
      type: 'asset/resource'  // Webpack 5 内置
    }
  ]
}
```

**Plugin(插件)**:

```js
const HtmlWebpackPlugin = require('html-webpack-plugin')

plugins: [
  new HtmlWebpackPlugin({
    template: './src/index.html'
  }),
  new MiniCssExtractPlugin({
    filename: '[name].[contenthash].css'
  })
]
```

### 2. Loader vs Plugin

| | Loader | Plugin |
|---|---|---|
| **作用** | 转换文件(TS→JS/Sass→CSS) | 打包优化/资源管理/环境变量注入 |
| **时机** | 模块加载时 | 打包全流程 |
| **写法** | 导出函数 | 导出类(有 apply 方法) |

**自定义 Loader**:

```js
// my-loader.js
module.exports = function(source) {
  // source = 文件内容
  return source.replace(/console\.log/g, '')  // 移除 console.log
}

// webpack.config.js
{
  test: /\.js$/,
  use: './my-loader.js'
}
```

**自定义 Plugin**:

```js
class MyPlugin {
  apply(compiler) {
    compiler.hooks.emit.tapAsync('MyPlugin', (compilation, callback) => {
      // 在生成文件前执行
      console.log('打包完成')
      callback()
    })
  }
}

plugins: [new MyPlugin()]
```

### 3. 打包流程

```
1. 初始化参数(配置+命令行)
   ↓
2. 初始化 Compiler(负责整个编译)
   ↓
3. 加载 Plugin(注册钩子)
   ↓
4. 开始编译
   ↓
5. 确定 Entry
   ↓
6. 编译模块(从 Entry 递归)
   - 调用 Loader 转换
   - 解析依赖(import/require)
   - 递归处理依赖
   ↓
7. 完成模块编译(得到依赖图 Dependency Graph)
   ↓
8. 输出资源(根据 Entry/依赖生成 Chunk)
   ↓
9. 写入文件系统
```

**核心对象**:
- **Compiler**:全局唯一,整个编译生命周期
- **Compilation**:每次编译创建,包含当前编译资源
- **Chunk**:代码块,多个模块组合
- **Module**:单个模块

### 4. 优化配置

**代码分割**(Code Splitting):

```js
optimization: {
  splitChunks: {
    chunks: 'all',
    cacheGroups: {
      vendor: {
        test: /[\\/]node_modules[\\/]/,
        name: 'vendors',
        priority: 10
      },
      common: {
        minChunks: 2,
        priority: 5,
        reuseExistingChunk: true
      }
    }
  }
}
```

**Tree-shaking**(摇树优化):

```js
// 前提:ESM + production mode
optimization: {
  usedExports: true,  // 标记未使用导出
  minimize: true,     // 压缩移除未使用代码
  sideEffects: false  // package.json 需配置
}

// package.json
{
  "sideEffects": false  // 或 ["*.css", "*.scss"]
}
```

**缓存**:

```js
cache: {
  type: 'filesystem',  // 持久化缓存(Webpack 5)
  buildDependencies: {
    config: [__filename]  // 配置文件变化清缓存
  }
}
```

**多进程**:

```js
// thread-loader
{
  test: /\.js$/,
  use: [
    'thread-loader',  // 多进程
    'babel-loader'
  ]
}
```

**压缩**:

```js
const TerserPlugin = require('terser-webpack-plugin')

optimization: {
  minimizer: [
    new TerserPlugin({
      parallel: true,  // 多进程
      terserOptions: {
        compress: {
          drop_console: true  // 移除 console
        }
      }
    })
  ]
}
```

### 5. HMR(热模块替换)

```js
devServer: {
  hot: true,
  port: 3000
}

// 代码中接受更新
if (module.hot) {
  module.hot.accept('./module.js', () => {
    // 模块更新时执行
  })
}
```

**原理**:
1. Webpack 监听文件变化
2. 重新编译变化的模块
3. 通过 WebSocket 推送更新
4. 客户端接收,替换模块
5. 不刷新页面,保持状态

## 三、Vite

### 1. Vite vs Webpack

| 特性 | Webpack | Vite |
|------|---------|------|
| **开发模式** | 打包所有模块 | 按需编译(ESM) |
| **启动速度** | 慢(全量打包) | 极快(无需打包) |
| **HMR** | 较慢 | 极快(精确更新) |
| **生产构建** | Webpack | Rollup |
| **配置** | 复杂 | 简洁 |
| **生态** | 成熟 | 快速发展 |

### 2. 核心原理

**开发模式**:

```
浏览器请求
   ↓
<script type="module" src="/src/main.js">
   ↓
Vite 拦截请求
   ↓
按需编译(esbuild)
   ↓
返回 ESM 模块
```

**依赖预构建**:

```js
// vite.config.js
optimizeDeps: {
  include: ['lodash-es'],  // 强制预构建
  exclude: ['your-package']
}
```

**为什么预构建**:
1. CommonJS → ESM
2. 多文件模块合并(如 lodash-es 600+ 模块 → 1 个)
3. 减少请求数

### 3. 基础配置

```js
// vite.config.js
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'
import react from '@vitejs/plugin-react'

export default defineConfig({
  plugins: [vue()],
  
  resolve: {
    alias: {
      '@': '/src'
    }
  },
  
  server: {
    port: 3000,
    proxy: {
      '/api': {
        target: 'http://localhost:8000',
        changeOrigin: true,
        rewrite: path => path.replace(/^\/api/, '')
      }
    }
  },
  
  build: {
    outDir: 'dist',
    rollupOptions: {
      output: {
        manualChunks: {
          vendor: ['vue', 'vue-router']
        }
      }
    }
  }
})
```

### 4. 插件生态

```js
import legacy from '@vitejs/plugin-legacy'
import compression from 'vite-plugin-compression'

plugins: [
  vue(),
  // 兼容旧浏览器
  legacy({
    targets: ['defaults', 'not IE 11']
  }),
  // Gzip 压缩
  compression()
]
```

### 5. 环境变量

```bash
# .env.development
VITE_API_URL=http://localhost:8000

# .env.production
VITE_API_URL=https://api.example.com
```

```js
// 使用(必须 VITE_ 前缀)
console.log(import.meta.env.VITE_API_URL)
```

## 四、Rollup

### 1. 特点

- **纯 ESM**:天然 Tree-shaking
- **输出纯净**:无多余运行时代码
- **适合库打包**:React/Vue 都用 Rollup

**配置**:

```js
// rollup.config.js
export default {
  input: 'src/index.js',
  output: [
    {
      file: 'dist/bundle.cjs.js',
      format: 'cjs'  // CommonJS
    },
    {
      file: 'dist/bundle.esm.js',
      format: 'esm'  // ESM
    },
    {
      file: 'dist/bundle.umd.js',
      format: 'umd',
      name: 'MyLib'
    }
  ],
  plugins: [
    resolve(),  // 解析 node_modules
    commonjs(), // 转换 CommonJS
    babel(),
    terser()    // 压缩
  ]
}
```

## 五、高频面试题

### Q1: Webpack 打包流程?

1. **初始化**:读配置,创建 Compiler
2. **编译**:从 Entry 递归,Loader 转换
3. **构建依赖图**:解析 import/require
4. **生成 Chunk**:根据 Entry/动态导入分割
5. **输出文件**:写入 dist

### Q2: Loader 执行顺序?

**从右往左,从下往上**:

```js
{
  test: /\.css$/,
  use: ['style-loader', 'css-loader', 'sass-loader']
  // 执行顺序:sass-loader → css-loader → style-loader
}
```

### Q3: Tree-shaking 原理?

1. **ESM 静态分析**:编译时确定导入/导出
2. **标记未使用**:`usedExports: true`
3. **压缩移除**:Terser 移除 dead code

**条件**:
- 使用 ESM(`import/export`)
- `production` mode
- `sideEffects: false`(package.json)

### Q4: 如何优化 Webpack 构建速度?

**减少搜索范围**:

```js
resolve: {
  modules: [path.resolve(__dirname, 'node_modules')],  // 指定查找路径
  extensions: ['.js', '.jsx'],  // 减少尝试后缀
  alias: {
    '@': path.resolve(__dirname, 'src')  // 别名
  }
}

module: {
  rules: [
    {
      test: /\.js$/,
      include: path.resolve(__dirname, 'src'),  // 只编译 src
      exclude: /node_modules/
    }
  ]
}
```

**缓存**:

```js
cache: {
  type: 'filesystem'  // Webpack 5 持久化缓存
}

{
  test: /\.js$/,
  use: [
    {
      loader: 'babel-loader',
      options: {
        cacheDirectory: true  // Babel 缓存
      }
    }
  ]
}
```

**多进程**:

```js
{
  test: /\.js$/,
  use: ['thread-loader', 'babel-loader']
}
```

**DllPlugin**(预编译第三方库):

```js
// webpack.dll.config.js
new webpack.DllPlugin({
  name: '[name]_[hash]',
  path: path.join(__dirname, 'dll', '[name]-manifest.json')
})

// webpack.config.js
new webpack.DllReferencePlugin({
  manifest: require('./dll/vendor-manifest.json')
})
```

### Q5: Vite 为什么快?

**开发模式**:
- **无需打包**:按需编译(ESM)
- **esbuild 预构建**:Go 语言,极快
- **精确 HMR**:只更新变化的模块

**生产模式**:
- **Rollup 打包**:Tree-shaking 更好

## 六、实战配置

### 场景1:多页面应用

```js
// webpack.config.js
const glob = require('glob')
const HtmlWebpackPlugin = require('html-webpack-plugin')

const entries = {}
const htmlPlugins = []

glob.sync('./src/pages/**/index.js').forEach(file => {
  const name = file.match(/pages\/(.*)\/index\.js/)[1]
  entries[name] = file
  htmlPlugins.push(
    new HtmlWebpackPlugin({
      template: `./src/pages/${name}/index.html`,
      filename: `${name}.html`,
      chunks: [name]
    })
  )
})

module.exports = {
  entry: entries,
  plugins: htmlPlugins
}
```
