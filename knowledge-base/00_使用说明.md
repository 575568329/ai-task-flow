# 知识库使用说明

欢迎使用你的个人知识库!这是一个**纯静态、离线可用**的文档浏览器,用来记录和阅读与 AI 讨论的各种方案。

---

## 项目说明

### 目的

记录与 AI 讨论的各种方案与笔记,解决两个痛点:

1. **随处可用**:项目下载/clone 下来,本地能直接看;也能托管到 GitHub Pages,任何设备打开网址就能访问。
2. **告别裸看 md**:不想直接打开 md 文件看难受的源码格式。打开 `index.html` 就能看到左侧目录树 + 右侧渲染好的文档,md/pdf/word/图片都能直接预览。

### 设计思路

- **左侧目录树 + 右侧预览**的经典文档站布局,目录树自动扫描 `docs/` 生成。
- **一套代码,两种场景通吃**:本地双击启动 / GitHub Pages 在线访问,前端都读同一份 `manifest.json` 渲染目录,逻辑统一。
- **离线优先**:第三方库全部本地化打包进项目,不依赖 CDN,断网也能用。
- **设计规范**:色彩 / 间距 / 圆角 / 阴影借鉴成熟设计 Token 体系,支持亮 / 暗主题。

### 关键技术取舍

| 问题 | 方案 |
|------|------|
| 浏览器 `file://` 安全限制读不了本地文件 | 用 Windows 自带 PowerShell(`HttpListener`)起零依赖本地服务,无需装 Node/Python |
| 本地与 GitHub Pages 目录如何统一 | 本地启动时 PowerShell 扫描、推送时 GitHub Actions 扫描,都生成同款 `manifest.json` |
| PowerShell 5.1 中文乱码 | 脚本存为 UTF-8 with BOM;生成的 manifest 存为 UTF-8 无 BOM |

### 技术栈

- **目录树 / 路由 / 搜索 / 主题**:原生 JS 手写,无框架
- **Markdown**:marked.js + highlight.js(代码高亮)+ mermaid(图表)
- **PDF**:PDF.js
- **Word(.docx)**:docx-preview(仅 .docx,复杂排版还原度有限)
- **图片**:原生 `<img>`
- **本地服务**:PowerShell + .NET HttpListener
- **自动部署**:GitHub Actions → GitHub Pages

### 项目结构

```
KnowledgeBase/
├── index.html              # 入口
├── 启动.bat                # Windows 一键启动
├── manifest.json           # 目录清单(自动生成)
├── docs/                   # 你的文档放这里
├── assets/
│   ├── css/                # 设计 token + 样式
│   ├── js/                 # app.js(主逻辑)+ viewers.js(预览器)
│   └── vendor/             # 本地化的第三方库
├── scripts/
│   ├── generate-manifest.ps1   # 扫描 docs/ 生成清单
│   └── serve.ps1               # 本地静态服务器
└── .github/workflows/pages.yml # 自动部署
```

## 怎么添加文档

把文件直接丢进 `docs/` 目录即可,支持的格式:

| 格式 | 扩展名 | 说明 |
|------|--------|------|
| Markdown | `.md` | 推荐格式,支持代码高亮、表格、Mermaid 图表 |
| PDF | `.pdf` | 逐页渲染 |
| Word | `.docx` | 仅支持新版 .docx(不支持老的 .doc) |
| 图片 | `.png` `.jpg` `.svg` 等 | 直接预览 |

### 用文件夹分组

在 `docs/` 下建子文件夹,左侧目录树会自动按文件夹分组。例如:

```
docs/
├── 前端方案/
│   ├── 组件库选型.md
│   └── 状态管理对比.md
└── 后端方案/
    └── 数据库设计.md
```

### 控制排序

文件按名称排序。想手动控制顺序,可以加数字前缀,显示时会自动隐藏:

- `01_概述.md` → 显示为「概述」
- `02_详细设计.md` → 显示为「详细设计」

## 两种打开方式

### 本地(Windows)

双击项目根目录的 **`启动.bat`**,它会:
1. 自动扫描 `docs/` 生成目录清单
2. 启动一个本地小服务(用 Windows 自带的 PowerShell,无需安装任何东西)
3. 自动打开默认浏览器

> 为什么不能直接双击 `index.html`?浏览器出于安全限制,`file://` 方式打不开本地的 md/pdf 文件。`启动.bat` 起一个本地服务就能绕开这个限制,Chrome/Edge 都能正常用。

### 在线(GitHub Pages)

推送到 GitHub 后,通过 Actions 自动部署,任何设备打开网址即可访问。详见根目录 `README.md`。

## 小技巧

- 左下角按钮可切换**亮色/暗色主题**
- 顶部搜索框可按文件名快速查找
- 右上角图标可在新标签页打开原始文件
