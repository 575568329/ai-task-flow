# claude-code-transcripts 项目详解

## 项目目标
将 Claude Code 会话文件（JSON 或 JSONL 格式）转换为整洁、移动端友好的分页 HTML 页面。

## 核心功能

**主要能力：**
- 转换本地和在线 Claude Code 会话为可浏览的多页面 HTML 文档
- 自动检测 GitHub 仓库并生成提交链接
- 支持发布到 GitHub Gist 获取分享链接
- 创建包含所有会话的可浏览存档
- 交互式会话选择器

**注意：** `web` 命令目前存在问题（issue #77），因为依赖的非官方 API 发生了变化。

## 技术栈
- **Python** (54.8%) - 主要开发语言
- **HTML** (42.0%) - 输出模板
- **JavaScript** (3.2%) - 交互增强
- **uv** - 包管理和运行工具

## 安装方式

```bash
# 使用 uv 安装
uv tool install claude-code-transcripts

# 或直接运行（无需安装）
uvx claude-code-transcripts --help
```

## 使用方法

**快速开始：**
```bash
claude-code-transcripts  # 交互式选择本地会话
```

**通用选项：**
- `-o, --output DIRECTORY` - 指定输出目录
- `-a, --output-auto` - 根据会话 ID 自动命名子目录
- `--repo OWNER/NAME` - 指定 GitHub 仓库用于生成提交链接
- `--open` - 在浏览器中打开生成的页面
- `--gist` - 上传到 GitHub Gist 并获取预览 URL
- `--json` - 在输出目录包含原始会话文件

## 可用命令

### 1. **local** (默认)
从 `~/.claude/projects` 选择本地会话
```bash
claude-code-transcripts local --limit 20
```

### 2. **web**
通过 Claude API 导入在线会话
```bash
claude-code-transcripts web SESSION_ID --gist
claude-code-transcripts web --repo simonw/datasette
```

### 3. **json**
转换特定的 JSON/JSONL 文件
```bash
claude-code-transcripts json session.json -o output-directory/
```

### 4. **all**
转换所有本地会话为存档
```bash
claude-code-transcripts all --open
claude-code-transcripts all --dry-run  # 预览不实际创建
```

## 生成的输出

**文件结构：**
- `index.html` - 包含提示和提交时间线的索引页
- `page-001.html`, `page-002.html` 等 - 分页的会话内容
- 可选包含原始 JSON 文件（使用 `--json` 选项）

**all 命令的存档结构：**
- 主索引列出所有项目
- 每个项目的会话列表页
- 各个会话的完整文档

## 特色功能

1. **Gist 发布** - 自动上传到 GitHub Gist，获取 gisthost.github.io 预览链接
2. **仓库过滤** - 按 GitHub 仓库筛选会话列表
3. **自动检测** - 自动识别 GitHub 仓库信息
4. **批量转换** - 使用 `all` 命令处理所有会话
5. **灵活输出** - 支持临时目录、自定义目录、自动命名等多种模式
